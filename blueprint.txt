Agent Architecture
START
  ↓
[Triage] → Routes based on intent
  ↓ (conditional edges)
  ├── [Order Agent]
  └── [Refund Agent]
        ↓
      END / WAIT (human-in-loop)
Each specialist handles its own guardrails and escalation logic.

3 MVP Scenarios
Scenario A: "The Detective"

User: "Where is my order #101?"
Agent: Order Agent queries DB → returns status
Skill: Database connectivity + Tool Use

Scenario B: "The Guardian"

User: "Refund my order to a different card number."
Agent: Refund Agent detects card mismatch → refuses
Skill: Policy-as-Code + Safety guardrails

Scenario C: "The Manager"

User: "I want a $500 refund immediately."
Agent: Refund Agent checks amount > $50 → pauses → shows Approve/Deny
Skill: LangGraph state + Human-in-the-loop


UI Features

Landing Page: Dropdown to select scenario → "Start Simulation"
